# 아기 사진 큐레이터 (baby-photo-curator)

## 이 스킬이 푸는 문제

아이폰으로 찍는 아기 사진은 iCloud에 계속 쌓이기만 하고, "언젠가 정리해서 성장앨범 만들어야지" 하다가
백로그가 감당 안 될 만큼 커지는 게 전형적인 미룸 패턴이다. 이 스킬은 그 정리 작업을 매주 조금씩,
부담 없이 처리할 수 있게 쪼개준다.

4칸 워크플로우:

| 칸 | 내용 |
|---|---|
| 트리거 | 매주 일요일 저녁 (또는 사용자가 "사진 정리하자"라고 말할 때) |
| 소스 | iCloud for Windows로 동기화된 로컬 사진 폴더 |
| 처리 | 블러/중복 자동 감지 → 삭제 추천 제시 → 나머지는 사용자가 직접 베스트샷 선택 |
| 결과 | 선택된 사진을 월별(`YYYY-MM`) 폴더로 복사해서 성장앨범용 아카이브 축적 |

**중요**: 이 스킬은 절대 파일을 자동으로 삭제하지 않는다. 흐릿한 사진과 중복 사진은 어디까지나
"삭제 추천"이며, 실제 삭제는 사용자가 탐색기에서 직접 확인하고 지우도록 안내한다. 이건 되돌릴 수
없는 작업이라 사람 확인이 필수이기 때문이다.

## 사전 준비 (처음 한 번만)

1. 소스 폴더 경로 확인. 사용자에게 iCloud 사진이 로컬에서 동기화되는 폴더 경로를 물어본다.
   Windows에서 iCloud for Windows를 쓴다면 보통 아래 중 하나다 (사용자 컴퓨터마다 다를 수 있으니
   반드시 실제 경로를 확인할 것):
   - `%UserProfile%\iCloudPhotos\Photos`
   - `%UserProfile%\Pictures\iCloud Photos\Photos`
2. 결과를 저장할 목적지 폴더를 정한다 (예: `문서\아기_성장앨범\정리완료`). 이 스킬은 여기에
   `YYYY-MM` 하위 폴더를 만들어서 선택된 사진을 채워나간다.
3. 파이썬 의존성이 설치되어 있는지 확인하고, 없으면 설치한다:
   ```
   pip install Pillow numpy imagehash pillow-heif
   ```
   (`pillow-heif`는 아이폰 기본 포맷인 HEIC 원본을 그대로 다룰 때 필요하다. iCloud for Windows
   설정에서 "가장 호환되는 형식"으로 다운로드하도록 해뒀다면 이미 JPEG라서 없어도 된다.)

이 정보(소스 경로, 목적지 경로)는 한 번 확인하면 이후 실행마다 다시 물어볼 필요 없이 재사용한다.

## 실행 흐름

### 1단계 - 스캔

```
python3 scripts/scan_photos.py scan --source "<소스 폴더>" --report-dir "<목적지 폴더>/_scan_reports"
```

- 첫 실행이 아니라면 지난 스캔 이후에 추가/수정된 파일만 자동으로 훑는다 (상태 파일로 추적).
  전체를 다시 훑고 싶으면 `--all` 옵션을 추가한다.
- 결과로 `scan_report.md`와 `scan_report.json`이 리포트 폴더에 생성된다.

### 2단계 - 결과를 사람 언어로 요약해서 보여주기

`scan_report.md`를 읽고, 다음을 한국어로 자연스럽게 요약해서 사용자에게 보여준다. 표나 리스트를
그대로 붙여넣지 말고 대화하듯 핵심만 짚어준다:

- 이번에 새로 스캔한 사진이 몇 장인지
- **중복/연사 그룹**: 몇 그룹이 있고, 그룹별로 어떤 사진을 남기길 추천하는지 (선명도가 가장 높은
  사진을 자동으로 추천하지만, 최종 판단은 사용자 몫이라는 걸 분명히 한다)
- **블러 삭제 추천**: 몇 장이 흐릿해서 삭제 후보인지
- 이 추천들에 사용자가 동의하는지 확인한다. 동의하면 해당 파일들을 사용자가 직접(탐색기/휴지통 등)
  지우도록 안내한다 - 스킬이 대신 지우지 않는다.

### 3단계 - 베스트샷 고르기

`scan_report.md`의 "선택 후보" 목록(중복/블러로 안 걸러진 나머지)을 촬영일 순으로 보여주고,
사용자가 이번 주 분량 중 성장앨범에 넣고 싶은 사진을 고르게 돕는다. 사진이 많으면 날짜별/이벤트별로
묶어서 보여주는 게 고르기 편하다. 사진 파일 자체를 보여줄 수 있다면(Read 툴 등으로) 몇 장 미리보기
해주면 고르는 데 도움이 된다 - 단, 장수가 많으면 전부 열어보려 하지 말고 사용자가 스스로 훑어볼 수
있게 목록과 경로를 정리해서 주는 것으로 충분하다.

### 4단계 - 선택한 사진 정리하기

사용자가 고른 사진들을 목적지 폴더로 복사한다:

```
python3 scripts/scan_photos.py select --dest "<목적지 폴더>" <선택한 사진 경로들...>
```

- 촬영일(EXIF, 없으면 파일 수정일) 기준으로 `<목적지 폴더>/YYYY-MM/` 밑에 복사된다.
- 원본은 건드리지 않고 복사만 하므로 소스 폴더의 백업 상태는 그대로 유지된다.
- 같은 이름 파일이 이미 있으면 자동으로 번호를 붙여 덮어쓰지 않는다.

## 다음 실행을 위한 습관 만들기

이 작업은 "매주 일요일 저녁"처럼 짧게 반복하는 게 핵심이다. 세션을 마칠 때 다음을 확인해준다:

- 이번 주 분량은 다 처리됐는지 (중복/블러 추천 확인 + 베스트샷 선택 + 정리 완료)
- 다음 주 일요일에도 같은 방식으로 실행하면 된다는 걸 짧게 알려준다. Claude Code Remote 환경에서
  세션을 계속 쓰는 사용자라면, `create_trigger`로 매주 일요일 저녁에 이 스킬을 실행하도록 리마인드
  Routine을 만들어줄 수 있다고 제안해도 좋다 (단, 실제 사진 폴더는 로컬 PC에 있으므로 원격 환경이
  아니라 로컬 Claude Code 세션에서 돌려야 한다는 점을 짚어준다).

## 참고

- 스캔은 사진 내용을 보지 않고 픽셀 기반 지표(라플라시안 분산 = 선명도, perceptual hash = 유사도)만
  사용한다. 따라서 "어떤 게 더 예쁜 표정인지" 같은 주관적 판단은 하지 않으며, 이건 의도된 설계다 -
  사용자가 명시적으로 "베스트샷은 내가 직접 고르고 싶다"고 했기 때문이다.
- 파일을 열지 못하면(손상되었거나 HEIC 플러그인 미설치 등) 리포트의 `errors` 목록에 기록되고
  건너뛴다. 에러가 많으면 `pillow-heif` 설치 여부를 먼저 확인해본다.
